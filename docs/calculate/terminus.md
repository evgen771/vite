## Изменение шрифта, размера, раскладки клавиатуры в TTY(консоль)

```bash
Terminus, доступны различные размеры: ter-v16n (обычный), ter-v24b (крупный жирный), ter-v32b (очень крупный)
n(norm) - нормальный
b(bold) - жирный
```
---

Для полноценной русификации консоли в `Gentoo (TTY)` необходимо настроить три компонента:   
шрифт с поддержкой кириллицы, раскладку клавиатуры для ввода и локаль системы для корректного отображения символов.

### Установка шрифта с кириллицей

Самый популярный шрифт с отличной поддержкой русского языка — `Terminus`.   
Чтобы он корректно отображал кириллицу, убедитесь, что пакет установлен:

`emerge media-fonts/terminus-font`

### Настройка вывода (шрифт)

Отредактируйте /`etc/conf.d/consolefont`.   
Для кириллицы в кодировке `UTF-8` лучше всего подходят шрифты с префиксом `ter-v` или `ter-u`

`nano /etc/conf.d/consolefont`

- consolefont="ter-v24b"  # Размер 24, жирный. Для HD мониторов можно ter-v32b.

*На моем компьютере*:

- consolefont="ter-v24n"   # n - нормальный

*Примечание: Если Terminus не подошел, можно использовать стандартный cyr-sun16* 

### Настройка ввода (раскладка)

Отредактируйте `/etc/conf.d/keymaps`, чтобы иметь возможность переключаться на русский язык. 
```bash
nano /etc/conf.d/keymaps
keymap="ru"             # Основная раскладка
extended_keymaps=""      # Можно оставить пустым или добавить дополнительные
windowkeys="yes"         # Использовать клавишу Win
```
Обычно переключение происходит по `Ctrl+Shift` или `Caps Lock`

### Настройка локали (UTF-8)

Для правильного отображения русского текста (например, в ls или mc) система должна использовать локаль `ru_RU.UTF-8` 

   1. Проверьте /etc/locale.gen, там должна быть раскомментирована строка:
   ru_RU.UTF-8 UTF-8
   2. Сгенерируйте локали: locale-gen
   3. Установите системную локаль: eselect locale set ru_RU.utf8

### Применение изменений

Чтобы всё заработало сразу и сохранялось после перезагрузки:

### Добавление в автозагрузку (если еще не добавлены)
```bash
rc-update add consolefont boot
rc-update add keymaps boot
```
### Перезапуск сервисов для применения настроек сейчас
```bash
rc-service consolefont restart
rc-service keymaps restart
```
**Важный нюанс: Если после всех настроек вместо русских букв видны «квадраты», проверьте, что в ядре включена поддержка UTF-8 в разделе Device Drivers -> Graphics support -> Console display driver support -> Framebuffer Console support**
